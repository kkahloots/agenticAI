## Key Concepts

### Deterministic vs Dynamic

**Deterministic (Version A)**:
- Fixed algorithm
- Direct tool calls
- No planning overhead
- Lower latency
- Lower observability

**Dynamic (Version B)**:
- Flexible strategy
- Registry-based tool discovery
- Explicit planning
- Higher latency
- Higher observability

### Agentic Features

**Planning**: Structured execution plans generated by agents
**Tool Discovery**: Tools discovered from registry, not hardcoded
**Memory**: Multi-layer memory system (plan, tool usage, observations)
**Agent Coordination**: Multiple agents working together
**Explainability**: Visible agent paths, tool selections, memory updates
**LLM Reasoning**: LLM used for decision-making and explanations
**Model Choice**: Explicit model selection based on context
**Fallback Strategy**: Graceful degradation when primary strategy fails

### Performance Tradeoffs

| Aspect | Deterministic | Dynamic |
|--------|---|---|
| Latency | Low (50-100ms) | Medium (150-300ms) |
| Memory | Minimal | 1-3KB per request |
| Throughput | High | Medium |
| Flexibility | Low | High |
| Observability | Low | High |

## Common Patterns

### Pattern 1: Deterministic + Dynamic Pair
Each UC has two versions:
- **A (Deterministic)**: Fixed algorithm, direct tool calls
- **B (Dynamic)**: Flexible strategy, registry-based tools, planning

### Pattern 2: Memory Integration
All dynamic UCs use memory:
- **Plan Memory**: Store generated plan
- **Tool Usage Memory**: Log tool calls and results
- **Observation Memory**: Store insights and learnings

### Pattern 3: Fallback Strategy
Dynamic UCs have fallback strategies:
- If primary strategy fails, fall back to deterministic
- If deterministic fails, fall back to simpler strategy
- Fallback tracked in memory

### Pattern 4: Agent Coordination
Dynamic UCs use multiple agents:
- **Orchestrator Agent**: Routes to appropriate UC
- **Strategy Agent**: Decides which strategy to use
- **Recommendation Agent**: Executes the strategy
- **Evaluation Agent**: Validates results
