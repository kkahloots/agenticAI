# Action Agent Specification\n\n## Overview\n\nThe Action Agent executes operations on external systems with comprehensive safety checks and validation. It handles CRM operations, notifications, case creation, and other system actions while ensuring security and compliance.\n\n## Implementation\n\n- **File**: `agentic/other_agents.py`\n- **Function**: `action_agent(state: AgenticState) -> dict`\n- **Autonomy Level**: Medium (makes safety-conscious execution decisions)\n\n## Core Responsibilities\n\n### 1. Action Execution\n- **CRM Operations**: Create, update, and manage customer records\n- **Notification Sending**: Send emails, SMS, and push notifications\n- **Case Management**: Create and update support cases\n- **Campaign Execution**: Launch marketing campaigns and promotions\n\n### 2. Safety Validation\n- **Permission Checks**: Verify user permissions for requested actions\n- **Risk Assessment**: Evaluate potential risks of actions\n- **Compliance Validation**: Ensure actions comply with regulations\n- **Data Protection**: Protect sensitive customer information\n\n### 3. Approval Workflows\n- **Risk-Based Approval**: Route high-risk actions for human approval\n- **Automated Approval**: Auto-approve low-risk actions\n- **Approval Tracking**: Track approval status and decisions\n- **Escalation Management**: Handle approval escalations\n\n### 4. Execution Monitoring\n- **Action Tracking**: Monitor action execution status\n- **Result Validation**: Verify action completion and success\n- **Error Handling**: Handle execution failures gracefully\n- **Rollback Capability**: Implement rollback for failed actions\n\n## MCP Server Integration\n\n### CRM Server\n- **Server ID**: `crm`\n- **Tools Used**:\n  - `create_case`: Create customer support cases\n  - `update_customer`: Update customer records\n  - `send_notification`: Send customer notifications\n  - `create_campaign`: Create marketing campaigns\n  - `schedule_followup`: Schedule follow-up actions\n\n### Customer Data Server\n- **Server ID**: `customer_data`\n- **Tools Used**:\n  - `get_customer_permissions`: Check customer communication preferences\n  - `validate_customer_identity`: Verify customer identity for sensitive actions\n\n## Decision Logic\n\n### Action Classification\n```python\ndef classify_action_type(request: str) -> str:\n    \"\"\"Classify the type of action based on the request.\"\"\"\n    request_lower = request.lower()\n    \n    # Notification actions\n    if any(keyword in request_lower for keyword in [\"send\", \"notify\", \"email\", \"sms\"]):\n        return \"notification\"\n    \n    # Case management actions\n    elif any(keyword in request_lower for keyword in [\"case\", \"ticket\", \"support\"]):\n        return \"case_management\"\n    \n    # Customer record actions\n    elif any(keyword in request_lower for keyword in [\"update\", \"modify\", \"change\"]):\n        return \"customer_update\"\n    \n    # Campaign actions\n    elif any(keyword in request_lower for keyword in [\"campaign\", \"promotion\", \"offer\"]):\n        return \"campaign_execution\"\n    \n    # Default to general action\n    else:\n        return \"general_action\"\n```\n\n### Risk Assessment\n```python\ndef assess_action_risk(action_type: str, parameters: dict, customer_data: dict) -> dict:\n    \"\"\"Assess the risk level of an action.\"\"\"\n    risk_score = 0.0\n    risk_factors = []\n    \n    # High-value customer actions are higher risk\n    if customer_data.get(\"segment\") == \"vip\":\n        risk_score += 0.3\n        risk_factors.append(\"vip_customer\")\n    \n    # Large monetary amounts increase risk\n    if \"amount\" in parameters and parameters[\"amount\"] > 1000:\n        risk_score += 0.4\n        risk_factors.append(\"high_value_transaction\")\n    \n    # Bulk actions are higher risk\n    if \"customer_count\" in parameters and parameters[\"customer_count\"] > 100:\n        risk_score += 0.3\n        risk_factors.append(\"bulk_action\")\n    \n    # Unverified identity increases risk\n    if customer_data.get(\"identity_status\") != \"verified\":\n        risk_score += 0.5\n        risk_factors.append(\"unverified_identity\")\n    \n    # Determine risk level\n    if risk_score >= 0.8:\n        risk_level = \"high\"\n    elif risk_score >= 0.5:\n        risk_level = \"medium\"\n    else:\n        risk_level = \"low\"\n    \n    return {\n        \"risk_level\": risk_level,\n        \"risk_score\": risk_score,\n        \"risk_factors\": risk_factors,\n        \"requires_approval\": risk_level == \"high\"\n    }\n```\n\n### Approval Decision\n```python\ndef determine_approval_requirement(risk_assessment: dict, action_type: str) -> dict:\n    \"\"\"Determine if action requires approval.\"\"\"\n    \n    # Always require approval for high-risk actions\n    if risk_assessment[\"risk_level\"] == \"high\":\n        return {\n            \"requires_approval\": True,\n            \"approval_type\": \"human\",\n            \"reason\": \"High risk action\"\n        }\n    \n    # Certain action types always require approval\n    high_risk_actions = [\"customer_update\", \"campaign_execution\"]\n    if action_type in high_risk_actions:\n        return {\n            \"requires_approval\": True,\n            \"approval_type\": \"human\",\n            \"reason\": f\"Action type '{action_type}' requires approval\"\n        }\n    \n    # Auto-approve low-risk actions\n    return {\n        \"requires_approval\": False,\n        \"approval_type\": \"automatic\",\n        \"reason\": \"Low risk action\"\n    }\n```\n\n## Action Execution Patterns\n\n### 1.
