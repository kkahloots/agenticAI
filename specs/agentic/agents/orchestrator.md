# Orchestrator Agent Specification\n\n## Overview\n\nThe Orchestrator Agent is the central coordinator of the agentic system. It serves as the entry point for all user requests, classifies intent, selects appropriate agents, and builds execution plans.\n\n## Implementation\n\n- **File**: `agentic/orchestrator_agent.py`\n- **Function**: `orchestrator_agent(state: AgenticState) -> dict`\n- **Routing Function**: `route_to_agent(state: AgenticState) -> str`\n\n## Core Responsibilities\n\n### 1. Intent Classification\n- **Primary Method**: LLM-based classification with structured JSON output\n- **Fallback**: Rule-based classification for reliability\n- **Pre-classification**: Fast pattern matching for unambiguous cases (SQL queries, etc.)\n\n### 2. Agent Selection\n- **Dynamic Routing**: Based on classified intent and context\n- **Confidence Thresholds**: Routes to human approval if confidence < 0.6\n- **Error Handling**: Routes to error handler on failures\n\n### 3. Execution Planning\n- **Plan Generation**: Creates structured execution plans per intent\n- **Resource Allocation**: Identifies required MCP servers and agents\n- **Step Definition**: Breaks down execution into logical steps\n\n### 4. Memory Context Loading\n- **Conversation History**: Loads session-based chat history\n- **User Profile**: Retrieves long-term user preferences\n- **Context Enrichment**: Provides agents with relevant background\n\n## Intent Classification\n\n### Supported Intents\n\n| Intent | Description | Target Agent | Confidence Threshold |\n|--------|-------------|--------------|---------------------|\n| `knowledge` | Information retrieval, factual lookup | knowledge_agent | 0.6 |\n| `analytics` | Data analysis, SQL queries, segmentation | analytics_agent | 0.6 |\n| `recommendation` | Personalized suggestions, ML recommendations | recommendation_agent | 0.6 |\n| `workflow` | Multi-step processes, complex goals | workflow_agent | 0.7 |\n| `action` | Execute operations, send notifications | action_agent | 0.8 |\n\n### Classification Logic\n\n#### 1. Pre-Classification Patterns\n```python\n# SQL queries - always analytics\nif re.search(r\"\\bselect\\b\", request.lower()) and re.search(r\"\\bfrom\\b\", request.lower()):\n    return \"analytics\", 0.95\n\n# Segmentation requests\nif any(w in request.lower() for w in [\"segment customers\", \"cluster customers\"]):\n    return \"analytics\", 0.95\n```\n\n#### 2. LLM Classification\n```python\nsystem_prompt = \"\"\"\nYou are an intent classifier for an agentic AI system.\nClassify the user request into one of these intents:\n- knowledge: factual lookup, information retrieval\n- analytics: data analysis, segmentation, SQL queries, insights, metrics\n- recommendation: personalized suggestions, next best action\n- workflow: multi-step processes, complex goals\n- action: execute operations, send notifications\n\nRespond with JSON: {\"intent\": \"<intent>\", \"confidence\": <0.0-1.0>, \"reasoning\": \"<brief explanation>\"}\n\"\"\"\n```\n\n#### 3. Rule-Based Fallback\n```python\n# Recommendation keywords\nif any(w in request.lower() for w in [\"recommend\", \"suggest\", \"next best\"]):\n    return \"recommendation\", 0.85\n\n# Knowledge keywords  \nif any(w in request.lower() for w in [\"what\", \"show\", \"retrieve\", \"find\", \"status\"]):\n    return \"knowledge\", 0.85\n```\n\n## Agent Selection Rules\n\n### Intent to Agent Mapping\n```python\nintent_to_agent = {\n    \"knowledge\": \"knowledge_agent\",\n    \"analytics\": \"analytics_agent\", \n    \"recommendation\": \"recommendation_agent\",\n    \"workflow\": \"workflow_agent\",\n    \"action\": \"action_agent\"\n}\n```\n\n### Special Routing Conditions\n- **Low Confidence** (< 0.6): Route to `human_approval`\n- **Error State**: Route to `error_handler`\n- **Unknown Intent**: Default to `knowledge_agent`\n\n## Execution Plans\n\n### Knowledge Plan\n```python\n{\n    \"agents\": [\"knowledge\"],\n    \"steps\": [\"Retrieve information\", \"Synthesize answer\"],\n    \"mcp_servers\": [\"customer_data\", \"product_catalog\"]\n}\n```\n\n### Analytics Plan\n```python\n{\n    \"agents\": [\"analytics\"],\n    \"steps\": [\"Analyze data\", \"Generate insights\"],\n    \"mcp_servers\": [\"analytics\"]\n}\n```\n\n### Recommendation Plan\n```python\n{\n    \"agents\": [\"recommendation\"],\n    \"steps\": [\"Load user profile\", \"Generate recommendations\", \"Rank results\"],\n    \"mcp_servers\": [\"recommendation\", \"customer_data\"]\n}\n```\n\n### Workflow Plan\n```python\n{\n    \"agents\": [\"workflow\", \"analytics\", \"action\", \"evaluation\"],\n    \"steps\": [\n        \"Decompose goal\",\n        \"Execute sub-tasks\", \n        \"Coordinate agents\",\n        \"Evaluate outcome\"\n    ],\n    \"mcp_servers\": [\"crm\", \"analytics\", \"customer_data\"]\n}\n```\n\n### Action Plan\n```python\n{\n    \"agents\": [\"action\"],\n    \"steps\": [\"Validate preconditions\", \"Execute action\", \"Record outcome\"],\n    \"mcp_servers\": [\"crm\", \"customer_data\"]\n}\n```\n\n## State Updates\n\n### Input State Requirements\n- `original_request`: User's request text\n- `user_id`: User identifier\n- `session_id`: Session identifier\n- `request_id`: Unique request ID\n\n### Output State Updates\n```python\n{\n    \"intent\": str,                    # Classified intent\n    \"confidence\": float,              # Classification confidence\n    \"active_agent\": str,              # Next agent to execute\n    \"agent_plan\": dict,               # Execution plan\n    \"agent_history\": [\"orchestrator\"], # Agent execution history\n    \"conversation_context\": dict,     # Loaded conversation history\n    \"user_profile\": dict,             # Loaded user profile\n    \"audit_trail\": [audit_entry]      # Audit log entry\n}\n```\n\n## Memory Integration\n\n### Memory Loading\n```python\n# Load conversation history\nconversation = memory_manager.conversation.get_conversation(session_id)\n\n# Load user profile\nuser_profile = memory_manager.user_profile.get_profile(user_id)\n```\n\n### Memory Updates\n```python\n# Record user message\nmemory_manager.conversation.add_message(session_id, \"user\", request)\n\n# Record orchestrator observation\nmemory_manager.agent_observation.record_observation(\n    \"orchestrator\",\n    f\"Classified intent as {intent} with confidence {confidence}\",\n    {\"request\": request, \"reasoning\": reasoning}\n)\n```\n\n## Audit Trail\n\n### Audit Entry Structure\n```python\n{\n    \"agent\": \"orchestrator\",\n    \"action\": \"classify_and_plan\",\n    \"intent\": intent,\n    \"confidence\": confidence,\n    \"next_agent\": next_agent,\n    \"timestamp\": datetime.now().isoformat()\n}\n```\n\n### Audit Logging\n```python\nlog_audit_event(\n    \"orchestrator_agent\",\n    \"classify_and_plan\",\n    {\"request\": request, \"intent\": intent, \"confidence\": confidence},\n    {\"plan\": plan, \"next_agent\": next_agent},\n    user_id=user_id,\n    request_id=state[\"request_id\"]\n)\n```\n\n## Error Handling\n\n### Classification Failures\n- **LLM Error**: Fall back to rule-based classification\n- **JSON Parse Error**: Use rule-based classification\n- **Unknown Intent**: Default to \"knowledge\" with low confidence\n\n### Memory Failures\n- **Conversation Load Error**: Continue with empty conversation context\n- **Profile Load Error**: Continue with empty user profile\n- **Memory Update Error**: Log error but continue execution\n\n## Performance Considerations\n\n### Optimization Strategies\n1. **Pre-classification**: Fast pattern matching before LLM call\n2. **LLM Caching**: Cache common intent classifications\n3. **Memory Batching**: Batch memory operations where possible\n4. **Async Operations**: Load memory context asynchronously\n\n### Latency Targets\n- **Pre-classification**: < 1ms\n- **LLM Classification**: < 500ms\n- **Memory Loading**: < 100ms\n- **Total Orchestrator Time**: < 600ms\n\n## Testing\n\n### Unit Tests\n- Intent classification accuracy\n- Agent selection logic\n- Plan generation correctness\n- Memory integration\n- Error handling\n\n### Integration Tests\n- End-to-end request routing\n- Memory persistence\n- Audit trail completeness\n- Error recovery\n\n### Test `ORCHESTRATOR_CONFIDENCE_THRESHOLD`: Minimum confidence for auto-routing (default: 0.6)\n- `ORCHESTRATOR_LLM_TEMPERATURE`: LLM temperature for classification (default: 0.0)\n- Size**: Maximum conversation history to load\n- **Classification Timeout**: Maximum time for LLM classification\n\n## Monitoring\n\n### Key Metrics\n- **Classification Accuracy**: Intent
