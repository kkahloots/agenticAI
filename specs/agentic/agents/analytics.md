# Analytics Agent Specification\n\n## Overview\n\nThe Analytics Agent handles data analysis, SQL query generation and execution, customer segmentation, and insights generation. It provides the analytical capabilities for the agentic system.\n\n## Implementation\n\n- **File**: `agentic/analytics_agent.py`\n- **Function**: `analytics_agent(state: AgenticState) -> dict`\n- **Autonomy Level**: Medium (makes analytical decisions)\n\n## Core Responsibilities\n\n### 1. SQL Query Processing\n- **Query Generation**: Convert natural language to SQL\n- **Query Execution**: Safe execution of read-only queries\n- **Result Processing**: Format and interpret query results\n- **Query Validation**: Ensure queries are safe and valid\n\n### 2. Customer Segmentation\n- **Segment Generation**: Create customer segments based on criteria\n- **Segment Analysis**: Analyze segment characteristics\n- **Dynamic Segmentation**: Real-time segment updates\n- **Segment Comparison**: Compare segments across metrics\n\n### 3. KPI Calculation\n- **Metric Computation**: Calculate business KPIs\n- **Trend Analysis**: Identify trends and patterns\n- **Performance Monitoring**: Track key performance indicators\n- **Benchmark Comparison**: Compare against historical data\n\n### 4. Insights Generation\n- **Pattern Detection**: Identify significant patterns in data\n- **Anomaly Detection**: Flag unusual data points\n- **Correlation Analysis**: Find relationships between variables\n- **Predictive Insights**: Generate forward-looking insights\n\n## MCP Server Integration\n\n### Analytics Server\n- **Server ID**: `analytics`\n- **Tools Used**:\n  - `run_sql_query`: Execute SQL queries safely\n  - `generate_segment`: Create customer segments\n  - `calculate_kpis`: Compute key performance indicators\n  - `analyze_trends`: Perform trend analysis\n  - `detect_anomalies`: Identify data anomalies\n\n## Decision Logic\n\n### Request Analysis Decision Tree\n\n```python\ndef determine_analytics_strategy(request: str) -> str:\n    request_lower = request.lower()\n    \n    # Direct SQL queries\n    if any(keyword in request_lower for keyword in [\"select\", \"sql\", \"query\"]):\n        return \"sql_execution\"\n    \n    # Segmentation requests\n    elif any(keyword in request_lower for keyword in [\"segment\", \"cluster\", \"group\"]):\n        return \"segmentation\"\n    \n    # KPI and metrics\n    elif any(keyword in request_lower for keyword in [\"kpi\", \"metric\", \"performance\", \"calculate\"]):\n        return \"kpi_calculation\"\n    \n    # Trend analysis\n    elif any(keyword in request_lower for keyword in [\"trend\", \"pattern\", \"over time\", \"historical\"]):\n        return \"trend_analysis\"\n    \n    # Default to general analysis\n    else:\n        return \"general_analysis\"\n```\n\n### SQL Safety Validation\n```python\ndef validate_sql_query(query: str) -> tuple[bool, str]:\n    \"\"\"Validate SQL query for safety and compliance.\"\"\"\n    query_upper = query.upper().strip()\n    \n    # Only allow SELECT statements\n    if not query_upper.startswith('SELECT'):\n        return False, \"Only SELECT queries are allowed\"\n    \n    # Block dangerous keywords\n    dangerous_keywords = ['DROP', 'DELETE', 'UPDATE', 'INSERT', 'ALTER', 'CREATE']\n    for keyword in dangerous_keywords:\n        if keyword in query_upper:\n            return False, f\"Dangerous keyword '{keyword}' not allowed\"\n    \n    # Require FROM clause\n    if 'FROM' not in query_upper:\n        return False, \"Query must include FROM clause\"\n    \n    return True, \"Query is safe\"\n```\n\n## Analytics Strategies\n\n### 1. SQL Query Execution\n\n#### Natural Language to SQL\n```python\ndef generate_sql_from_natural_language(request: str) -> str:\n    \"\"\"Convert natural language request to SQL query.\"\"\"\n    # Use LLM to generate SQL\n    system_prompt = \"\"\"\n    You are a SQL expert. Convert the natural language request to a safe SELECT query.\n    Only use these tables: customers, interactions, products, campaigns.\n    Always include appropriate WHERE clauses and LIMIT results to 100.\n    \"\"\"\n    \n    llm = get_llm(temperature=0.0)\n    response = llm.invoke([\n        SystemMessage(content=system_prompt),\n        HumanMessage(content=f\"Generate SQL for: {request}\")\n    ])\n    \n    return extract_sql_from_response(response.content)\n```\n\n#### Query Execution\n```python\n# MCP call to analytics server\nresult = analytics_server.invoke_tool(\n    \"run_sql_query\",\n    query=sql_query,\n    limit=100\n)\n\n# Expected result structure\n{\n    \"query\": \"SELECT * FROM customers WHERE segment = 'vip'\",\n    \"results\": [\n        {\"customer_id\": \"CUST-001\", \"segment\": \"vip\", \"ltv\": 15000},\n        {\"customer_id\": \"CUST-002\", \"segment\": \"vip\", \"ltv\": 12000}\n    ],\n    \"row_count\": 2,\n    \"execution_time_ms\": 45\n}\n```\n\n### 2. Formatting\n```python\ndef format_sql_results(result: dict) -> str:\n    \"\"\"Format SQL query results for human consumption.\"\"\"\n    if not result.get(\"results\"):\n        return \"No results found.\"\n    \n    rows = result[\"results\"]\n    row_count = result.get(\"row_count\", len(rows))\n    \n    # Create summary\n    summary = f\"Found {row_count} results:\\n\"\n    \n    # Show first few rows\n    for i, row in enumerate(rows[:5]):\n        row_str = \", \".join(f\"{k}: {v}\" for k, v in row.items())\n        summary += f\"  {i+1}. {row_str}\\n\"\n    \n    if len(rows) > 5:\n        summary += f\"  ... and {len(rows) - 5} more rows\\n\"\n    \n    return summary\n```\n\n### Segmentation Result Formatting\n```python\ndef format_segmentation_results(result: dict) -> str:\n    \"\"\"Format segmentation results for human consumption.\"\"\"\n    segment_name = result.get(\"segment_name\", \"Unknown\")\n    customer_count = result.get(\"customer_count\", 0)\n    characteristics = result.get(\"segment_characteristics\", {})\n    \n    summary = f\"Segment '{segment_name}' created with {customer_count} customers:\\n\"\n    \n    if characteristics:\n        summary += \"Characteristics:\\n\"\n        for key, value in characteristics.items():\n            summary += f\"  - {key}: {value}\\n\"\n    \n    return summary\n```\n\n### KPI Result Formatting\n```python\ndef format_kpi_results(result: dict) -> str:\n    \"\"\"Format KPI results for human consumption.\"\"\"\n    kpis = result.get(\"kpis\", {})\n    time_period = result.get(\"time_period\", \"unknown period\")\n    \n    summary = f\"KPIs for {time_period}:\\n\"\n    \n    for metric_name, metric_data in kpis.items():\n        value = metric_data.get(\"value\")\n        change_pct = metric_data.get(\"change_pct\", 0)\n        trend = metric_data.get(\"trend\", \"stable\")\n        \n        change_indicator = \"↑\" if change_pct > 0 else \"↓\" if change_pct < 0 else \"→\"\n        summary += f\"  - {metric_name}: {value} ({change_indicator} {abs(change_pct):.1f}%, {trend})\\n\"\n    \n    return summary\n```\n\n## State
