# 🧪 Customer Data Generation Rules

**TL;DR**: Rules for generating 208 synthetic online-store shoppers that stress-test all agent paths — unverified identity, null fields, duplicate phones, multilingual names, and realistic segmentation.

Run: `python scripts/cust_dataset_generator.py --output data/customers.json --count 208`

---

### [Distribution Rule]

**Formula**: `IF field = identity_status THEN weights = [verified:0.60, unverified:0.30, pending:0.10]`

**Explanation**: 30% unverified identity ensures the re-verification path is exercised.

**Edge Cases**:
- null `identity_expiry_date` when `identity_status = "pending"` (never completed)
- `identity_expiry_date` in the past when `identity_status = "unverified"`

---

### [Segment Rule]

**Segments**: `dormant_vip` (15%), `at_risk` (20%), `new` (15%), `casual` (20%), `vip` (30%)

---

### [Multilingual Name Rule]

**Name pools**:
- `en`: e.g. James Carter, Sarah Mitchell, Oliver Brown
- `hu`: e.g. Kovács Péter, Nagy Eszter, Szabó Gábor (surname-first format)
- `ar`: e.g. Ahmed Kareem, Fatima Al-Rashid, Omar Khalil
- `zh`: e.g. 陈伟强, 王丽华, 张建国
- `el`: e.g. Νικόλαος Παπαδόπουλος, Σοφία Αναγνωστοπούλου

**Distribution**: ~34% EN, ~33% HU, ~33% AR (plus zh/el for new customers)

---

### [Null & Duplicate Rule]

- `~10%` null email
- `~5%` duplicate phone

---

### [Consent Rule]

- `consent_flags.marketing = false` for ~15% of customers
- `consent_flags.sms = false` for ~20%
- `consent_flags.data_sharing = false` for ~10%

---

### [Scores]

- `fraud_score`: uniform 0.0–1.0; null for ~5% → assign dataset mean at runtime
- `engagement_score`: correlated with segment — `dormant_vip` gets 0.1–0.3, `vip` gets 0.7–1.0
- `return_risk`: 0.0–0.4 for verified identity; 0.5–1.0 for unverified

---

### [Purchase Categories]

Available: `electronics`, `clothing`, `books`, `home`, `sports`, `beauty`, `toys`, `automotive`

Cross-sell triggers used by NBA scoring:
- `electronics` → `PROMO-PREMIUM-MEMBERSHIP`
- `home` → `PROMO-BUNDLE-DEAL`
- `clothing` → `PROMO-WINBACK`

---

### [Promotions]

| Code | Name | Target Segments | Cross-sell Trigger |
|------|------|----------------|--------------------|
| `PROMO-PREMIUM-MEMBERSHIP` | Premium Membership | vip, dormant_vip, at_risk | electronics |
| `PROMO-LOYALTY-POINTS` | Loyalty Points Boost | vip, new, at_risk | — |
| `PROMO-BUNDLE-DEAL` | Bundle Deal | vip, dormant_vip, new | home |
| `PROMO-WINBACK` | Win-Back Offer | at_risk, casual, dormant_vip | clothing |

---

### [Sales Transactions]

Generated by `scripts/generate_sales_data.py` → `data/sales_transactions.json`

- 898 transactions across 208 customers
- Date range: 2025-01-01 → 2026-03-31
- Channels: `website` (40%), `mobile_app` (30%), `social_media` (15%), `email_campaign` (10%), `affiliate` (5%)
- Statuses: `completed` (80%), `returned` (12%), `cancelled` (8%)
- Revenue by category: electronics dominant (~65%), sports_outdoors, home_garden, clothing follow
